# v1.3.4

发布日期：2026-03-15

## 重点更新

- 新增 v2 controller façade：`inspect / plan / apply / verify / recover / service / update / skills` 子命令入口。
- 状态目录新增 `latest-task.json`、`plans/`、`runs/`、`events.jsonl`，用于最近任务摘要、计划冻结与运行审计。
- 交互首页头部新增最近任务摘要展示，便于快速确认上一次执行结论。

## 控制器与只读语义修复

- `inspect footprint` 现在为严格只读路径：不再创建状态目录、锁文件或 `index.jsonl`，也不会触发 Zig 自动修复下载。
- `plan/apply/verify` 现在会冻结计划阶段匹配范围，并在 `apply` 前做一致性预检，避免动态重扫导致目标漂移。
- `space-governance` 显式 `--targets` 的 `verify` 已适配“目标已清理”场景，不再因目标不存在而误报失败。
- `controller-state` 更新运行记录时改为写入 `run_updated`，事件流审计语义恢复正确。

## CLI / 文档 / Agent 契约收口

- 公共 CLI 建议命令全部切到 v2 子命令，不再输出 `--upgrade` / `--sync-skills` 等旧入口。
- README、`docs/NON_INTERACTIVE_SPEC.md`、Agent skill 文档与命令参考已统一到 v2 契约。
- `update apply` 缺少确认时现在会明确提示 `--ack UPGRADE`。
